How this works

Fair price is our robust model: take the last 14 daily average prices, drop the current (live, in-progress) day, drop the highest & lowest couple of completed days, then take a recency-weighted average of the rest. It tracks real moves but a single buyout spike or troll listing can't sway it.

deal = (spot - fair) / fair (negative = below fair = a buy)

What "Liquid" means. The market API gives no trade volume, so we judge liquidity from price behaviour scaled by value (a stand-in for dollar volume): an item shows in the default view only if it's an actual material / honing / fusion / cooking item (we hide engraving recipes, consumables and baubles), it has a full recent price history, and its 14-day price is steady within a limit that grows with its price — a high-value item can be a bit jumpier and still count (each trade moves real gold), while a cheap item must trade tightly to qualify. A hard ceiling still drops wildly-spiky items at any price, so dead-tier-but-expensive stones don't sneak back in. Raise Min value to hide cheap, low-stakes items, or hit Everything to see all items the API tracks.

Caveats. Prices are a snapshot (auto-refreshed every few hours); we don't label per-unit vs. per-stack on every item, but the deal % is unit-independent. Hover a sparkline for the daily values.
